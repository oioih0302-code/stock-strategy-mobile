# 주식 전략 판단기 V1.3

자동매매가 아니라 **매수/매도 의사결정 보조 앱**입니다.  
사용자 스타일에 맞춰 `한 번에 매수`, `추가매수 없음`, `40%/40%/20% 분할매도`, `전략 저장 후 일관성 유지`를 기본 규칙으로 사용합니다.

## V1.2 추가 기능

- 관심종목 여러 개 동시 스캔
- 국장/미장 종목 수 제한 없음
- `KRX:005930`, `US:NVDA`처럼 시장을 섞어서 스캔 가능
- 점수순 후보 정렬
- 상위 후보를 바로 전략 저장 가능
- 캡쳐 이미지 OCR 자동입력
  - 종목코드 후보
  - 현재가/체결가 후보
  - 보유현금/예수금 후보
  - 수량/거래총액 후보
- OCR로 읽은 값을 전략 분석 탭, 현금 보정, 거래 기록에 바로 반영 가능

## 핵심 규칙

1. 종목 수 제한 없음
2. 신규 매수 예산은 기본 100만~110만원
3. 앱 기준 현금이 110만원 이상이면 105만원 기준 계산
4. 앱 기준 현금이 100만~110만원이면 가진 현금 안에서 계산
5. 추가매수/물타기 판단 없음
6. 매도는 40% / 40% / 20% 분할매도
7. 저장 전략이 있으면 기존 매수구간/손절가/추격금지선을 우선 적용
8. 무료 시세 데이터는 지연 또는 누락될 수 있으므로 실제 주문 전 증권앱 현재가 확인 필수


## Streamlit Cloud 배포 방법

1. 이 폴더 안의 파일을 GitHub 저장소에 업로드합니다.
2. Streamlit Community Cloud에서 새 앱을 만듭니다.
3. main file path를 `app.py`로 설정합니다.
4. 배포 후 폰 브라우저로 접속합니다.
5. 브라우저 메뉴에서 “홈 화면에 추가”를 누르면 앱처럼 사용할 수 있습니다.

## OCR 관련 주의

`packages.txt`에 아래 시스템 패키지를 넣었습니다.

```text
tesseract-ocr
tesseract-ocr-kor
```

Streamlit Cloud는 보통 `packages.txt`를 읽어 OCR 엔진을 설치합니다.  
단, OCR은 화면 품질에 따라 틀릴 수 있습니다. 특히 쉼표, 소수점, 종목코드, 체결금액은 반드시 직접 확인해야 합니다.

## 로컬 실행 방법

Python이 설치된 PC에서 실행할 경우:

```bash
pip install -r requirements.txt
streamlit run app.py
```

로컬에서 OCR을 쓰려면 Tesseract OCR도 별도로 설치해야 합니다.

## 사용 순서

### 1. 현금 맞추기

`현금/포트` 탭에서 증권앱에 보이는 보유현금을 입력합니다.

### 2. 관심종목 등록 또는 직접 입력

`관심종목 스캔` 탭에서 관심종목을 등록하거나 아래처럼 직접 입력합니다.

```text
KRX:005930
KRX:267260
US:NVDA
US:RKLB
```

### 3. 스캔 실행

같은 알고리즘으로 여러 종목을 분석하고 점수순으로 보여줍니다.

### 4. 캡쳐 자동입력

`캡쳐 자동입력` 탭에서 증권앱 캡쳐를 올립니다.
OCR 후보를 확인한 뒤 전략 분석, 현금 보정, 거래 기록에 반영합니다.

### 5. 전략 저장

매수 구간, 손절가, 목표가가 괜찮으면 전략을 저장합니다.
다음 분석부터 기존 전략과 비교해 근거 없는 가격 상향 조정을 막습니다.

## 한계

## V1.3 수정사항

- 미장 보유화면 캡쳐에서 `$` 표기가 있으면 시장을 US로 우선 인식합니다.
- `0.702803주` 같은 소수 수량을 `702803` 종목코드로 착각하는 문제를 줄였습니다.
- 미장 화면의 평가금액/투자원금을 보유현금으로 자동 반영하지 않도록 보수적으로 수정했습니다.
- 가격 후보는 `$182.42`처럼 달러 표기가 붙은 숫자를 우선 표시합니다.
